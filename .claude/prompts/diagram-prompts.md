# 项目文档图表生成提示词模板

## 快速提示词

### 1. 生成数据流图

```
请为当前Django项目生成详细的数据流图，要求：
1. 包含顶层数据流图（Level 0）和第一层数据流图（Level 1）
2. 展示用户、管理员与系统各模块之间的数据交互
3. 标明数据存储（数据库、Session等）
4. 使用Mermaid语法，保存到[输出目录]
5. 包含核心业务流程的数据流
```

### 2. 生成数据库ER图

```
请分析当前项目的所有models.py文件，生成数据库E-R图，要求：
1. 使用Mermaid erDiagram语法
2. 显示所有实体及其字段（标注PK/FK）
3. 显示实体间的关系基数
4. 提供简化版核心关系图
5. 保存到[输出目录]
```

### 3. 生成功能模块图

```
请为当前项目生成功能模块图，要求：
1. 分别生成用户端和管理员端的功能模块
2. 使用Mermaid mindmap语法
3. 包含系统架构模块图
4. 功能分类清晰，层级不超过3层
5. 保存到[输出目录]
```

### 4. 生成用例图

```
请为当前项目生成用例图，要求：
1. 分别生成用户端和管理员端的用例图
2. 使用Mermaid flowchart语法模拟UML用例图
3. 标注用例间的关系（include/extend）
4. 提供主要用例的说明表（前置条件、主要流程）
5. 保存到[输出目录]
```

### 5. 一键生成所有图表

```
请为当前Django项目生成完整的项目文档图表，包括：
1. 数据流图（详细版）
2. 数据库E-R图（简洁版）
3. 功能模块图（用户端+管理员端）
4. 用例图（用户端+管理员端）

要求：
- 分析项目的models.py和views.py
- 使用Mermaid语法
- 保存到桌面的"项目图表"文件夹
- 图表内容使用中文
```

## 详细提示词模板

### 模板A：完整项目分析

```
我需要为Django电商项目生成一套完整的技术文档图表。

## 项目背景
- 项目名称：[项目名称]
- 技术栈：Django + SQLite
- 主要功能：用户管理、商品浏览、购物车、订单管理

## 需要生成的图表

### 1. 数据流图
- 分析用户从浏览到下单的完整数据流
- 包含Session购物车的数据流
- 展示库存扣减的事务处理
- 输出：数据流图.md

### 2. 数据库ER图
- 分析所有数据模型及关系
- User <-> Profile <-> Address
- Category -> Product -> OrderItem -> Order
- Coupon, Review, Wishlist等
- 输出：数据库ER图.md

### 3. 功能模块图
- 用户端：账号、商品、购物车、订单、互动
- 管理员端：商品管理、订单管理、用户管理、营销
- 系统架构：展示层、业务层、数据层
- 输出：功能模块图.md

### 4. 用例图
- 列出所有用户用例（~20个）
- 列出所有管理员用例（~20个）
- 标注用例关系
- 提供用例规格说明表
- 输出：用例图.md

## 输出要求
- 格式：Markdown + Mermaid
- 语言：中文
- 位置：桌面/项目图表/
```

## 提示词使用技巧

### 1. 明确输出位置
始终指定输出目录，避免文件散落：
```
保存到桌面的"项目图表"文件夹
```

### 2. 指定详细程度
根据需求选择详细或简洁：
```
生成简洁版的ER图，只显示核心实体和关系
生成详细的数据流图，包含所有数据处理步骤
```

### 3. 分角色生成
分别生成用户端和管理员端的图表：
```
请分别生成用户端和管理员端的功能模块图
请分别生成普通用户和管理员的用例图
```

### 4. 迭代优化
先生成基础版本，再要求补充：
```
在现有数据流图中补充优惠券验证的流程
在ER图中增加CouponUsage实体
```

### 5. 格式转换
可以要求其他输出格式：
```
除了Mermaid格式，请提供PlantUML格式的ER图
请生成可以粘贴到Draw.io的XML格式
```

## 注意事项

1. **Mermaid渲染**：需要在支持Mermaid的查看器中打开（如Typora、VS Code + Mermaid插件、GitHub）

2. **图表更新**：代码变更后需要重新生成图表以保持同步

3. **语言选择**：可通过提示词指定使用中文或英文

4. **自定义需求**：可以要求添加项目特定的图表元素

5. **批量生成**：建议一次性生成所有图表，保证风格一致
